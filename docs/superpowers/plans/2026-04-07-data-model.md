# Redeemo Data Model Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Define and create the complete PostgreSQL schema for the Redeemo platform — all tables, relationships, constraints, and indexes — as Prisma schema files with a seed script for dev data.

**Architecture:** Single PostgreSQL database serving all four platform surfaces (customer app, customer website, merchant web portal, merchant mobile app, admin panel) via a shared API. Schema is managed with Prisma ORM. All tables use UUID primary keys. Soft deletes where data must be preserved (merchants, users, vouchers). Hard deletes only for join tables.

**Tech Stack:** Node.js 24, TypeScript, Prisma ORM 5.x, PostgreSQL 16, Neon (serverless PostgreSQL — no Docker required)

## Task 5: Write the voucher models

**Files:**
- Modify: `prisma/schema.prisma`

This is the most business-critical model group. Pay close attention to the state machine and the monthly cycle enforcement table.

**Voucher state machine:**
```
DRAFT → PENDING_APPROVAL → ACTIVE ↔ INACTIVE → EXPIRED
                        ↘ REJECTED (back to DRAFT for edit)
```

**Monthly cycle enforcement:**
`UserVoucherCycleState` — one row per (user × voucher). When a user's subscription renews (new `current_period_start`), the service layer resets `isRedeemedInCurrentCycle = false` for all vouchers where the merchant still has the voucher active. This table is the source of truth for "can this user redeem this voucher right now?"

- [ ] **Step 5.1: Append voucher models**

```prisma
// ─────────────────────────────────────────
// ENUMS (add to existing enums block)
// ─────────────────────────────────────────

enum VoucherType {
  BOGO               // Buy One Get One Free
  SPEND_AND_SAVE
  DISCOUNT_FIXED
  DISCOUNT_PERCENT
  FREEBIE
  PACKAGE_DEAL
  TIME_LIMITED
  REUSABLE
}

enum VoucherStatus {
  DRAFT
  PENDING_APPROVAL
  ACTIVE
  INACTIVE
  EXPIRED
}

enum ApprovalStatus {
  PENDING
  APPROVED
  REJECTED
}

enum ValidationMethod {
  PIN        // merchant 4-digit passcode entered by customer
  QR_SCAN    // QR code scanned by branch staff
  MANUAL     // Quick Validate by merchant admin (no code)
}

// ─────────────────────────────────────────
// VOUCHERS
// ─────────────────────────────────────────

model Voucher {
  id               String         @id @default(uuid())
  merchantId       String
  code             String         @unique // RMV-001 or RCV-001 format
  isMandatory      Boolean        @default(false)
  type             VoucherType
  title            String
  description      String?
  terms            String?        // custom terms text
  imageUrl         String?
  estimatedSaving  Decimal        @db.Decimal(10, 2) // in GBP
  expiryDate       DateTime?      // null = no hard expiry
  status           VoucherStatus  @default(DRAFT)
  approvalStatus   ApprovalStatus @default(PENDING)
  approvalComment  String?        // admin rejection reason
  approvedAt       DateTime?
  approvedBy       String?        // AdminUser.id
  publishedAt      DateTime?
  createdAt        DateTime       @default(now())
  updatedAt        DateTime       @updatedAt

  merchant         Merchant       @relation(fields: [merchantId], references: [id])
  redemptions      VoucherRedemption[]
  cycleStates      UserVoucherCycleState[]
  favouritedBy     FavouriteVoucher[]

  @@index([merchantId])
  @@index([status])
  @@index([approvalStatus])
  @@index([code])
}

// ─────────────────────────────────────────
// VOUCHER REDEMPTION (the event record)
// ─────────────────────────────────────────

model VoucherRedemption {
  id               String           @id @default(uuid())
  userId           String
  voucherId        String
  branchId         String           // which branch it was redeemed at
  validatedById    String?          // BranchUser.id — null if customer self-redeemed via PIN
  validationMethod ValidationMethod
  estimatedSaving  Decimal          @db.Decimal(10, 2) // snapshot at time of redemption
  redeemedAt       DateTime         @default(now())

  user             User             @relation(fields: [userId], references: [id])
  voucher          Voucher          @relation(fields: [voucherId], references: [id])
  branch           Branch           @relation(fields: [branchId], references: [id])
  validatedBy      BranchUser?      @relation(fields: [validatedById], references: [id])

  @@index([userId])
  @@index([voucherId])
  @@index([branchId])
  @@index([redeemedAt])
  @@index([userId, voucherId]) // fast lookup for cycle state check
}

// ─────────────────────────────────────────
// USER VOUCHER CYCLE STATE
// The source of truth for: "can user X redeem voucher Y right now?"
//
// Logic:
// - One row per (user × voucher)
// - isRedeemedInCurrentCycle = true → user cannot redeem again this cycle
// - When subscription renews (new period_start):
//     UPDATE UserVoucherCycleState
//     SET isRedeemedInCurrentCycle = false, cycleStartDate = new period_start
//     WHERE userId = ? AND voucher is still ACTIVE
// ─────────────────────────────────────────

model UserVoucherCycleState {
  id                      String   @id @default(uuid())
  userId                  String
  voucherId               String
  cycleStartDate          DateTime // the subscription period_start this cycle maps to
  isRedeemedInCurrentCycle Boolean @default(false)
  lastRedeemedAt          DateTime?

  user                    User     @relation(fields: [userId], references: [id], onDelete: Cascade)
  voucher                 Voucher  @relation(fields: [voucherId], references: [id], onDelete: Cascade)

  @@unique([userId, voucherId])  // one row per user × voucher pair
  @@index([userId])
  @@index([isRedeemedInCurrentCycle])
}
```
